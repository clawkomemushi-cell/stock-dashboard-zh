# 買賣操作同步方案（讓蝦米蟲也讀得到）

## 目前狀態

現在網站上的「今天的買賣操作」是存在瀏覽器的 localStorage。

這代表：
- 米蟲在手機或電腦上輸入後，網站自己看得到
- 但蝦米蟲目前**不能直接讀到**這些資料
- 所以現在還不能真正拿來做：
  - 提醒提早跑
  - 對照是否已持有
  - 自我反省推薦是否正確

## 目標

做到：
- 米蟲在網站輸入買賣操作
- 資料會被存到一個蝦米蟲也讀得到的地方
- 蝦米蟲後續可用來：
  - 讀取持倉/操作紀錄
  - 在分析網站上顯示
  - 做自我反省
  - 做提醒或提早出場建議

## 最推薦方案

### 網站 → Webhook / 小 API → JSON 儲存

流程：
1. 米蟲在網站輸入操作
2. 網站把資料 POST 到一個 webhook / API
3. webhook / API 把資料存成 JSON
4. 蝦米蟲讀取該 JSON
5. 網站也可讀取相同 JSON 顯示歷史紀錄

## 為什麼推薦這條

因為：
- GitHub Pages 是純靜態網站，前端不能安全直接寫 GitHub repo
- token 不能放前端
- localStorage 又只有米蟲的裝置自己看得到

所以需要一個「中間層」來安全收資料。

## 可選實作路徑

### 方案 A：Cloudflare Worker + JSON / KV

優點：
- 很適合接 GitHub Pages
- 成本低
- 可做簡單 API
- 後續擴充方便

缺點：
- 需要 Cloudflare 帳號與一些設定

### 方案 B：Google Apps Script + Google Sheet

優點：
- 很快能做出來
- 輸入資料後很好檢視
- 蝦米蟲也容易讀取

缺點：
- 結構感比較不像正式系統
- 之後若要做更完整 JSON 流程，要再轉一層

### 方案 C：自架小型 API（Node / Python）

優點：
- 最自由
- 可完全客製

缺點：
- 維護成本較高
- 要有穩定主機

## 目前建議順序

1. 先完成前端輸入格式（已進行）
2. 再做最小可用同步方案
3. 優先考慮：
   - Google Apps Script（最快 MVP）
   - 或 Cloudflare Worker（比較乾淨）
4. 最後再把資料接進：
   - 股票網站顯示
   - 自我反省
   - 提醒 / 提早跑邏輯

## 如果要真正實用，之後還要補的欄位

建議買賣資料至少包含：
- 日期時間
- 買進 / 賣出
- 代號
- 名稱
- 價格
- 交易單位（股 / 零股 / 張）
- 數量
- 備註（為什麼買 / 賣）
- 是否已成交

## 下一步建議

如果要我繼續實作，建議下一步是：

### 先做 MVP
- 用 Google Apps Script 做一個接收表單資料的 API
- 網站把操作資料 POST 過去
- 存進 Google Sheet
- 蝦米蟲再讀 Google Sheet

這會是目前最快能讓「米蟲輸入 → 蝦米蟲看得到」的路。
